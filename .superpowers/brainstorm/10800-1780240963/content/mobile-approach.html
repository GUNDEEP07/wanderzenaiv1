Recommended Mobile Strategy

Mobile-first, not mobile-adapted. Every pattern below treats 375px as the default, desktop as the enhancement.

RULE 01 — Navigation
Logo left · CTA right · nothing else
No hamburger menu. On a travel SaaS with 5 pages, a hamburger adds interaction cost for almost no gain. The only mobile nav action that matters is "Plan my trip." Every nav link is secondary — show them only at ≥768px.
  • Nav height: 52px (touch-friendly)
  • Logo + wordmark left, CTA pill right
  • Links hidden below 768px, visible above
  • Consistent across every single page

RULE 02 — Hero / Landing
Full-width CTAs · 2×2 stats grid · no decorative tags
Buttons go full-width on mobile — thumbs reach the full width. Stats become a 2×2 grid instead of 4 horizontal items. Destination tags are hidden (already done). Background grid/glow effects: keep them.
  • Primary CTA: full width, 52px tall
  • Ghost CTA: full width, below primary
  • Stats: 2-column grid, not horizontal strip
  • Hero title: clamp(2.2rem, 8vw, 6.5rem) — already good

RULE 03 — Content Sections
Single column · scrollable tab rows · full-width cards
Every multi-column grid collapses to single column on mobile. Horizontal tab bars (Explore page continents, Dashboard filters) get overflow-x scroll with touch momentum. Cards go edge-to-edge with 16px horizontal padding.
  • Photo strip: 2-column grid (not 6) on mobile
  • Continent tabs: horizontal scroll, no wrap
  • Pricing cards: single column
  • Feature cards: single column
  • Section padding: 4rem 1rem (not 7rem 2rem)
